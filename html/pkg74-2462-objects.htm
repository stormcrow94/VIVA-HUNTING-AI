schedule (object)

Installation schedule of a policy package to devices.
When adding schedule in workspace mode, the ADOM should be committed first.

Supported methods: get, add, delete

Object URL:
Table object/pm/pkg/global/<pkg_name_path>/schedule
/pm/pkg/adom/<adom_name>/<pkg_name_path>/schedule
Attribute ListDescriptions
adom_rev_commentstring (127 bytes)
The comment for the new ADOM revision. It will be ignored in add request if adom_rev_name is not set.
adom_rev_namestring (35 bytes)
The name for the new ADOM revision. If not set in add request, no ADOM revision will be created during installation.
datetimestring (20 bytes)
Indicates the date and time of the schedule.
It should follow the following format: "YYYY-MM-DD hh:mm".
The value of this attribute should not be earlier than the current time of the system.
dev_rev_commentstring (127 bytes)
The comment for the device configuration revision that will be generated during install.
scopescope object, refer to device object, or group object
Target device.
In the get request, it includes only devices.
In the add request:
  • It can be set to both device and device group.
  • The devices/device groups should be in the package installation target list.
  • If no target is provided in the request, the installation will be scheduled for all package installation targets.
  • If no VDOM is provided for a device target, the installation will be scheduled for all VDOMs of the device in the package installation list.
